# SNSスクショから「買いたい」を整理するアプリ【SCshotOkaimono(仮)】

## このアプリについて

SNSで見かけた「これ欲しい！」という商品をスクショしたのに、  
気づけばカメラロールに埋もれてしまい、  
いざ買い物のタイミングでは「なんだったっけ？」となる――  

このアプリは、そんな “スクショ難民問題” を解決するための  
**「文脈も含めて“欲しい”を整理する買い物メモアプリ」** です。

スクリーンショットをアップロードすると、  
AIがその画像を解析し、

- 商品名
- SNS投稿者名
- 投稿内容の要約（なぜ魅力的だったか）
- 参考価格（投稿時点の価格）
- 購入候補ショップと商品ページ候補
- 商品画像サムネイル

などを抽出・整理し、**あとから迷わず思い出して“買える状態”**に整えます。

---

## 解決したい課題

* SNSで見つけた商品をスクショしても、時間が経つと埋もれてしまい、思い出せない
* 「なぜ欲しいと思ったのか」という **文脈を忘れてしまう**
* いいね / 保存 / 外部共有 など、SNS内ブックマークだけでは管理しきれない
* SNS投稿者が紹介していた情報を見返したいのにURLがなくて辿れない
* 「本気で買いたいもの」と「一旦気になっただけ」が混ざって整理できない

---

## どう解決するか

このアプリでは、次のような仕組みでこれらの課題を解決します。

### 1. スクショ → 「意味のある整理済みデータ」に変換

スクショをアップロードすると、  
OpenAI API による解析で次の情報を抽出します。

- SNS投稿者名
- SNSプラットフォーム（Instagram / X / TikTok など）
- 商品名
- 投稿内容の要約（魅力ポイント・ベネフィット）
- 投稿内で示されている価格（あれば）
- 商品カテゴリ候補
- 購入候補ショップの情報
- 商品イメージ（外部のOGP画像などから取得）

スクリーンショットそのものは長期保存せず、 
解析後は必要なテキスト・メタ情報のみを保存する方針です。

---

### 2. SNS依存しない「欲しい理由」の見える化

SNSに戻らなくても、

- どんな内容の投稿だったか
- どこが良いと言っていたのか
- どんなベネフィットがあったか

が要約テキストとして残るため、  
「なんでこれ欲しかったんだっけ？」問題を解消します。

---

### 3. 商品名からショップ候補とURLを提示

抽出した商品名を元に検索を行い、

- 公式サイト
- ECサイト（楽天 / Amazon / 他）
- 有力購入候補

を候補として提示します。

自動で一択に決め打ちするのではなく  
ユーザーが採用する情報を選べる設計を想定しています。

---

### 4. 価格は「登録時の参考価格」として扱う

価格は変動するため、

- “登録時に見た価格” を参考価格として保存
- 現在価格保証はしない

という現実的なスタンスで管理します。

---

### 5. 商品イメージをサムネイルとして表示

- 商品ページのOGP画像
- ショッピングAPIの画像

などを用いて、  
リスト画面でも商品のビジュアルが分かる設計を目指します。

---

### 6. 「なんとなく」と「本気」を分けて管理

各アイテムには次のいずれか1つのステータスを持たせます。

* 気になる
* 買いたい
* 購入済み
* 不要だった

登録時のデフォルトステータスは「気になる」とし、  
ユーザーが必要に応じて「買いたい」「購入済み」「不要だった」へ切り替えられる仕様を想定しています。  
ステータスは単一選択制とし、ワンタップで切り替えできるシンプルな操作を目指します。

---

## ユーザー層について

- SNSから商品を知ることが多い人
- 「とりあえずスクショだけして放置」になりがちな人
- いいねや保存機能だけでは整理しきれない人
- “なぜ欲しかったのか” まで含めて記録したい人
- 楽天ROOMやアフィリエイトリンク経由に抵抗がある人

---

## 想定する利用フロー

1. **日常**
   SNS閲覧中、「これ欲しい！」と思った商品をいつも通りスクショ

2. **まとめて整理**
   スキマ時間にスクショをまとめてアップロード

3. **AIが整理**
   - 商品名抽出
   - SNS投稿者名取得
   - 投稿内容要約
   - 候補ショップ提示
   - サムネ生成

4. **必要な情報だけ確認して保存**
   - タイトルだけ設定でもOK
   - 価格は「投稿時価格」として記録

5. **買い物時に使う**
   「買いたい」だけ絞り込み  
   → URLから商品ページへ  
   → 購入後は「購入済み」へ変更

## MVP

まずはアプリの核となる価値である  
「スクショを“意味のある買い物リスト”へ変換する」ことを最優先とします。

* ユーザー登録 / ログイン（メールアドレス + パスワード）
* スクリーンショット画像アップロード
* OpenAI APIによる解析
  * SNS投稿者名抽出
  * 商品名抽出
  * 投稿内容の要約
* ステータス管理（単一選択）
* 「買いたい」抽出ビュー
* 検索結果画像
* 購入候補ショップ提示（候補型）

## 他サービスとの違い

* 単なるメモ・ToDoではなく  
  **「SNS由来の“欲しい”に特化した整理ツール」**
* スクショ依存なのに  
   **画像を保存しない安全設計**
* URLがなくても  
   **“文脈ごと記録されるから思い出せる”**
* ショップ候補提示＋サムネ表示で  
   **買い物直前の実用性に強い**

本アプリは、画像認識や商品検索を行う既存サービスとは目的と体験が異なります。

Googleレンズなどの画像検索ツールは、画像をもとにその場で情報を調べることはできますが、
「あとで買うために覚えておく」「買い物の場面で迷わず探す」といった用途には最適化されていません。

また、SNS標準の保存機能やコレクション機能は、SNSを閲覧している最中にしか活用しづらく、
実店舗での買い物には向いていません。

Google Keep などのメモアプリでは、商品名や店名を自分で入力する必要があり、
スクリーンショットは単なる画像として溜まってしまいがちです。

本アプリでは、SNSで撮りためたスクリーンショットを後から解析し、
商品名・店候補・価格などを自動で補助入力することで、
「撮っただけで終わらない」整理体験を提供します。

さらに、商品を「買える店」ごとにリスト化できるため、
無印良品やニトリなどの実店舗に行った際にも、
効率よく商品を探し、確認できる点が特徴です。

他のアプリと似ているように見えても、機能の組み合わせとしては日本市場で未だ提供されていない
独自体験を目指しています。既存のウィッシュリストアプリや買い物メモアプリは
手入力や共有機能が中心で、自動的にスクリーンショットから商品情報や店舗候補を抽出し、
「買い物」という行動の前後に最適化された体験を提供するものはありません。
また、LIKEtoKNOW.itのような海外サービスとも異なり、日本ブランドや実店舗の利用シーンまで
視野に入れた設計になっています。

## 技術スタック（想定）

* Ruby on Rails 7.x
* Tailwind CSS
* MySQL 8.x
* JavaScript
* OpenAI API（Vision & テキスト解析）
* OGP取得 / もしくはショッピングAPI
* ActiveStorage（解析中のみの一時利用・長期保存なし）

---

## 方針について

- スクリーンショット画像は解析処理のため一時的に利用し、長期的な保存は行わない
- URLは自動確定ではなく「候補提示型」
- 情報精度より“使える整理体験”を重視
